This section contains prompts to help you publish and deploy your FHIR Implementation Guide. Use these prompts with AI assistants to ensure successful publication and deployment of your IG. Ultimately for publishing an LLM code assistant can likely help run and publish your IG successfully. However there are some publishing tasks it might still not be able to completely do for you like publishing your guide on https://build.fhir.org/ or a GitHub Pages site.

Assumptions & Core Principles

Assumptions

  • Default to FHIR R4 unless otherwise stated
  • Validate outputs with SUSHI and IG Publisher
  • Use canonical URLs for IGs, profiles, extensions, and value sets
  • Prefer small, iterative prompts over large, monolithic ones

Three Core Prompting Principles

  1. Be Descriptive — clearly state the goal and provide examples/context
  2. Be Specific — scope the task tightly; avoid vague requests
  3. Break It Down — split complex tasks into smaller steps

LLM Integration Tips

  • Start simple and increase complexity iteratively
  • Reuse the same chat/session to preserve context
  • Provide samples (FSH, JSON, text) to shape outputs
  • Always validate LLM outputs with your tooling and specs

I recommend Elliot Silver's guide on How to Publish a FHIR Implementation Guide to learn how to publish a GitHub guide. Also check out his blog on setting up webhooks to get your CI Build (web hosted build) up and running.

💡 Tips for Using These Prompts

  1. Customize the prompts by replacing the bracketed text with your specific details
  2. Provide context about your IG components, publication goals, and deployment requirements
  3. Ask follow-up questions to dive deeper into specific publication areas
  4. Save successful prompts for future use and share them with the community
  5. Iterate and refine prompts based on the responses you receive
